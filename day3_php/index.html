switch_case_default 匹配判断


//使多种代码块根据某一条件执行代码
定义赋值

case 赋值1:

输出;
break;

循环结构?

在给定条件范围内，指定代码块重复执行
//判断结束边界类似于布尔运算true/false决定循环体是否继续执行

for循环


//以,分隔条件表达1：初始化条件定义多个
for(条件表达1:开始初始化(定赋值=); 条件表达2:结束判定边界(true/false); 条件表达3:操作式){
输出;
}

或

初始化：定义赋值 //条件表达1
for(:条件表达2:结束判断边界(true/false);条件表达3:操作式){
输出;
}

实现原理：

  1. 执行一次：条件表达1 开始初始化
  2. 执行N次：条件表达2 结束循环体判断边界(true/false)
  3. 执行N次：输出(循环体)
  4. 执行N次：条件表达3 进入自操作变化
  5. 重复循环342


while循环


定义赋值; //条件表达1:开始初始化(定赋值=)
while(条件表达2:结束判断边界(true/false)){
输出(循环体);
条件表达3:操作式 //循环条件的变更;
}

do_while循环


//先做一次执行再判断 区别于while(){}先判断后执行
定义赋值; //条件表达1:开始初始化(定赋值=)
do{
//输出(循环体)
if(){
//条件表达3:操作式;
}
}while(条件表达2:结束判断边界(true/false));

文件加载：数据包含

在php脚本中拉取使用另一php脚本的代码，共同开发php功能实现代码共享

包含实现形式

  1. 向上索取：先定义后索取使用输出 //可重复解释文件：通常用于流程控制
    1. include “”; //包含页定义 输出页包含展示输出：作为定义赋值((定义)输出多次执行多次)
  3. 2 include_once “”:; //包含页定义包含展示 输出页输出：作为输出((定义)输出多次执行一次)
  4. 向下给予： //区别于报错形式：代码执行较严谨
    1. require “”;
    1. require_once “”:;

      php代码实现逻辑

  7. 读取：在代码段中读取程序代码
  8. 编译：把代码段中代码转入数据段以字节存储 即 生成opcode
  9. 解析：使用zendengine引擎解析代码opcode后在栈区以二进制执行
  10. 输出可读写代码：堆区回收内存

    包含实现原理

  11. 执行文件加载时会把包含文件中代码嵌入到当前文件
  12. 文件包含位置即为当前文件加载位置
  13. 文件包含中代码作为php单独编译执行：包含代码段被输出执行时才报错

    文件加载路径

    绝对路径：相对于系统磁盘/
    相对路径：相对于当前文件夹./

函数function ?(,){}

作为一本基本语法结构，将某功能代码块封装到结构中实现功能的复用

函数语法结构

function关键字 函数名? 参数$/“”：形参,/实参 函数体 return返回值

  1. 为实现函数复用，一个功能一个函数
  2. 函数参数在调用时需包含参数
  3. 函数定义与调用没有顺序要求：执行时在内存址中找到对应函数

函数执行实现逻辑

先编译后再执行

  1. 读取代码块
  2. 编译代码转到数据段(内存)以字节存储
  3. 栈区以二进制逐行执行
  4. 堆区回收

函数命名：字母/数字/下划线

驼峰法：AaBbCc();
下划线法：a_b_c();

函数参数()

形式参数：命名无意义
实际参数：调用实际参与内存址执行 //变量 有值表达式(变量/常量/运算符值)
//调用函数时：形参是实参的载体，需要在函数内部找到实参内存址的对应位置，将实参内存址上数据传值给形参赋值后执行代码；
//形参实参不一定同名；

形参默认值

指形参在没有调用实参内存址参与执行时的默认定义赋值输出；
独立存在：

  1. 仅在函数内部执行不与外部实参变量冲突；
  2. 内部形参与外部实参重名不冲突；

参数执行的实现逻辑：

  1. 调用函数：寻找函数内存址是否存在
  2. 栈区执行：代码段转入数据段后作为索引开辟栈区执行
  3. 判断形参是否存在；调用的实参是否存在
  4. 执行函数体：调用实参内存址数据值传值给形参
  5. 执行返回函数值

参数传递模式：引用传值&$

实参调用时通过引用传递传值给形参；形参与外部传入赋值无关联；
引用传值&$：函数内部形参从外部实参获取数据时，要明确&$才会使内部函数主动获取外部内存址，之后外部赋值才会绑定内部形参同步运算
关于绑定： //内部运算引用传值绑定外部内存址

  1. 函数内部形参指向绑定外部内存址，之后会同步内外赋值；
  2. 引用传值不得传入实体数据值，只有变量可以使用引用传值(变量存在内存址)

函数体{}

实现：定义变量/常量/流程控制/调用函数

返回值return 作为结束符

函数实现返回结果给外部调用：默认返回空值NULL
代码结束符：函数体内终止函数；函数体外终止代码；

定义作用域：变量常量

指能够被访问的区域

  1. 全局变量：函数外部 用户自定义
    所属空间：仅在全局空间访问，函数体内不可访问全局变量；区别于js可在函数体内访问全局空间变量 //脚本周期为代码结束
    //函数内部访问输出不了全局变量
  2. 局部变量：函数体内部
    所属空间：允许函数本身内部访问 //函数周期为函数结束(栈区开辟独立内存空间)
    //全局空间访问输出不了局部变量
  3. 超全局变量：系统自定义 预定义
    所属空间：无访问限制
    //局部变量访问全局空间：
    1. 内部通过超全局$GLOBALS[“数组”]方式访问全局变量
    2. 形实参数传值
    3. 引用传值&$
    4. global $关键字?：实现内外(函数体内空间/外部全局空间)变量跨域互访；特殊使用时；
      global $?;
      $? = ?;
      1. global定义赋值在外部存在且同名，内部定义赋值指向外部内存址；
      2. global定义赋值在外部不存在，内部绑定外部自动生成外部同名内存址；

函数类型

静态变量static?

在函数内部声明实现定义赋值多次调用时：初始化一次，不被内存回收；
//即编译时自行生成定义赋值存在后多次调用初始化一次：依据上一次结果赋值进行运算(后自增赋值)；
//通常用于：统计函数调用次数：统筹函数多次递归调用的结果；
static $? = ?;

可变函数：回调?

当定义赋值为函数名时，调用定义赋值同名()可调用函数()
//通常为外部自定义函数调用系统函数时使用；
$? = ?函数;
echo ?();

匿名函数

定义赋值为没有名字的函数；实际存储值为对象

闭包函数?

闭包closure是函数体内部变量中，内层被定义赋值(匿名函数)引用绑定未释放所形成
//保留外层函数定义赋值给内部函数定义赋值(匿名函数)使用
use($外层定义赋值)
