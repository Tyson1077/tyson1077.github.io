mysql数据类型

计算机数据单位：1byte比特 = 8bit位

列字段类型

int整型

  1. tinyint()迷你整形：保存一字节8位，最大表示数值255/-127到128；

    //最大表示三位数值时为0-255；
    //最大表示四位数值时为负127-正128；
  2. smallint()小整形：保存两字节16位，最大表示数值65535；
  3. mediumint()中整形：保存三字节24位；
  4. int()标准整型：保存四字节32位；
  5. bigint()大整型：保存八字节64位；

    列属性：无负数符号标识unsigned

    无符号又称无负数标识，存储数据到当前字段没有负数表示；
    alter table myint add int_6 tinyint unsigned first;
    insert into myint values(255,100,255,255,255,255);

    列属性：类型尾椎数()zerofill

    表示当前数据类型最多显示长度；
    即显示长度可自定：超出长度以默认最值表示显示，不足长度可自定显示范围

    //通过指定列属性unsigned后，tinyint迷你整型数据最多表示三位长度；

    //指定列属性zerofill左侧填充0补进数位；(自动同步添加无标识属性)

    //指定列属性长度最少表示显示范围 数据类型(?)；

    小数数据型

  6. 浮点型：有失精度类型；

    基本语法： //数据类型
    float():不指定浮点小数位；
    float(整数位?,小数位?):分开存储整数小数位；


//四舍五入处理非法数据；

//无法存储指定非法数据，通过四舍五入处理自动存储；

//支持科学计数法?e?；
1.1 float()单精浮点：单精度存储4字节数据，最大保留7位精度；
1.2 double()双精浮点：双精度存储8字节数据，最大保留15位精度；

  1. 定点型：有失小数精度，保证整数精度；
    decimal()定点型：4字节存储9位数动态分配，分开存储整数小数位；
    基本语法： //数据类型
    decimal(整数位65?,小数位30?) //整数位无法四舍五入最值；

    时间日期型

  2. date日期类型：3字节存储数据；
    格式：yyyy-mm-dd 年月日
    范围：1000-01-01到9999-12-12

  3. time时间类型：3字节存储数据；
    格式HH:ii:ss 时分秒
    范围：-838:59:59到正838:59:59
    //表示时间点或时间区间；

  4. datetime日期时间型：8字节存储合并日期时间数据；
    格式：YYYY-mm-dd HH:ii:ss 年月日时分秒
    范围：1000-01-01 00:00:00到9999-12-12 23:59:59

  5. timestamp时间戳型：保留格林威治时间；
    格式：YYYY-mm-dd HH:ii:ss 年月日时分秒
    //对应数据被手动修改会自动更新为系统生成时间戳数据；

  6. Year年类型：单字节存储数据；
    格式：抽象0-99年或具体四位年 抽象简写/具体年
    范围：1900-2155
    //匹配两位抽象简写年时，小于69匹配20/大于70匹配19；

    字符串型

  7. char()定长字符：自动分配非空值空间指定字节存储数据；
    格式：char(?)
    范围：0-255

  8. varchar()变长字符：根据实际数据字节分配空间存储数据；
    格式：varchar(?)
    范围：0-65535理论上

  9. text()文本类型：存储普通字符文本；
    3.1 tinytext()：单字节存储数据，实际存储2^8+1；

  10. 2 text()：双字节存储数据，实际存储2^16+2；

  11. 3 mediumtext()：三字节存储数据，实际存储2^24+3；

  12. 4 longtext()：四字节存储数据，实际存储2^32+4；

  13. blob()二进制文本类型：存储二进制文本，通常以链接指向文本本身；

  14. enum()枚举类型：手动指定预设值存储数据； //单选
    格式：enum(预设数据值?,预设数据值?,预设数据值?)
    范围：单字节存储255数据值/两字节存储65535数据值；

    存储原理：存储数据并非预设数据值，而是系统自动生成始于1的下标值对应存储元素数据值；

  15. set()集合型：同时保存多个数据存储； //复选
    格式：set(数据值1,数据值2,数据值n)

    //通过多个数据值选项自动生成存储多个字节空间；
    存储原理：

    1. 同enum()枚举类型类似；
    2. 实际数据存储值为系统自动生成的下标值对应0/1占位值；
    3. 实际存储值查看表示为0/1占位值颠倒转换成十进制的和；
    4. 以数值方式查看集合数据值；
    5. 可通过数据值方式代位添加存储数据项；

      ##mysql记录最值长度
      记录最值长度为varchar()长度；

  16. mysql记录长度为record == 行row；

  17. 存储结构：数据存储空间+记录大小长度空间；

  18. 理论记录最值为65535；utf8存储记录最值为21844，gbk存储记录最值为32766；

  19. 不同字符集占用多个字节不同；

    列字段属性

    [空值操作] [列描述] [字段默认值] [额外属性]

NULL空值属性

代表字段为空值；
//判断操作是否允许该字段为空值：能为空/不能为空；
mysql记录长度最值为65535字节：

  1. 若表字段属性为NULL则实际存储数据长度大小=65534；
  2. 保留一字节存储NULL空值；
    选项：
  3. NULL
  4. NOT NULL

    default预设默认值

    默认条件下用户不指定操作数据时的预设数据值：通常为NULL；
    //预设该字段必选默认值；

    添加：预设default默认值

    create table 表名?(
    字段? 列类型? NOT NULL, –非空值
    预设字段? 预设列类型? default ? –用户没插入数据值则使用预设默认值?
    )charset utf8;

    使用default预设默认值

    insert into 表? (字段?) values(“数据值?”);

    insert into 表? values(“数据值?”,default);

    comment列描述

    为开发人员提供注释说明；

    添加：comment列描述

    create table 表名?(
    字段? 列类型? NOT NULL comment “列描述?”
    )charset utf8;

    查看：comment列描述

    通过表结构创建命令：查看列描述；
    show create table 表名?;

    KEY主键

    primary key主键指表中有且有字段中有唯一数据值； //不允许重复；

    创建主键

  5. 随表创建
  6. 1 主键字段后+primary key属性关键字
  7. 2 所有字段后+primary key属性选项:primary key(主键字段?)
  8. 表后增加
    通过修改表字段属性：新增字段列属性；

    主键约束

  9. 字段数据不为空；
  10. 字段数据不重复；

    主键分类

    指主键对应字段的业务实现意义；
  11. 业务主键：主键所在字段拥有业务意义；
  12. 逻辑主键：自然增长的整型确保唯一意义；

    EXTRA额外属性：自动增长auto_increment

    当字段没有确定数据值时获得自动增长属性，系统根据前面既存数据进行逻辑增长填充数据值；

    原理

  13. 定义符合自动增长属性字段的维护数据值以及原始步长；
  14. 用户新增数据时没指定数据值，则系统自动通过维护数据值加上步长生成新数据值； //按指定规律递归操作；
  15. 触发自动增长：给定属性字段无数据值；

    创建自增长

    //随表创建自增长：auto_increment属性关键字；
    create table atc(
    id int primary key auto_increment,
    name varchar(10) not null comment “user”,
    pswd varchar(10) not null comment “password”
    )charset utf8;

    //通过修改表字段属性：新增字段列属性；
    alter table 表名? add auto_increment;

查看自增长：触发

插入空数据触发； //表中自增长有且只有一个；
insert into 表名? values(null空数据值?,”数值1”,”数值2”);

修改自增长

通过修改表选项结构：修改自增长的值；
alter table 表名? auto_increment = 自增长值?;

删除自增长

修改字段自增长属性后系统自动删除表中自增长；
alter table 表名? modify 自增段字段? 字段类型?;

自增长初始设置

  1. 自增长变量分类
    auto_increment_increment步长值；
    auto_increment_offset初始值；
  2. 查看初始变量：自增长
    show variables like “auto_increment%”;
  3. 修改初始变量：自增长
    set auto_increment_increment = ?; //修改步长；
    set auto_increment_offset = ?; //修改初始值；

    自增长基本规则

  4. 表中有且只有一个自增长字段属性； //表选项限定绑定；
  5. 数据插入未触发自增长则需要手动新增自增长属性；
    alter table 表名? modify 自增长字段 字段类型 auto_increment;
  6. 用户数据值优先级大于自增长的数据值；

    表关系

    指表之间实体存在的关系；

    一对一关系

    表中数据与另表数据有且只有匹配一条关系；
    设计思路：两表中使用相同主键；

    一对多关系：一对多/多对一

    在多关系表中维护(新增插入)作为主键的单一关系字段；
    设计思路：在多关系中维护指定的主键单一关系字段；

    多对多关系

    在多表中多条数据可以匹配多条数据，反之相同；
    设计思路：通过第三表数据记录两张多表的联系数据记录；
